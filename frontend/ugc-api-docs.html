🎬 UGC Creator API · v1.0

Build AI-powered
video editors with one API

Upload a raw video. Get back a professionally edited, captioned, B-roll overlaid, music-synced vertical short — fully automated, ready for social media.

🔒 Auth: X-App-Token header
📦 Format: multipart/form-data + JSON
🚀 Output: MP4 (1080×1920)
⏱️ Processing: 2–5 min per video
🔑 Authentication

Every request must include your API token in the X-App-Token header. Get your token from the dashboard under Settings → API Keys.

⚠️
Keep your token private. Never expose it in client-side JavaScript or public repositories.
X-App-Token: your_api_token_here

📤 Upload Video

Upload a raw video file to create a new UGC job. Returns a job_id to use in subsequent calls.

POST /api/ugc/upload Upload raw video file ▾
Content-Type
multipart/form-data
Max File Size
100 MB
ℹ️
Accepted formats: .mp4, .mov, .avi, .mkv. For best results, use MP4 with H.264 encoding.

Request Body (form-data)

FieldTypeRequiredDescription
fileFileRequiredThe raw video file to upload.

Response (200 OK)

{ "success": true, "job_id": "a1b2c3d4e5f6g7h8", "filename": "my_video.mp4", "message": "Video uploaded successfully." }
⚙️ Process Video

Trigger the AI video enhancement pipeline with your chosen feature flags. Processing runs asynchronously — poll for status after calling this.

POST /api/ugc/process/{job_id} Start AI enhancement pipeline ▾
Content-Type
application/json
Processing Time
2–8 minutes

Path Parameters

ParamTypeRequiredDescription
job_idstringRequiredThe job_id returned from the upload endpoint.

Request Body (JSON)

FieldTypeDefaultDescription

subtitle_stylestring"default"Caption animation style. See Subtitle Styles section.
brollbooltrueAI-generated B-roll image overlays at key moments.
broll_sourcestring"pollinations""pollinations" = auto-generate via AI. "meta_ai" = use pre-uploaded images. "pexels" = use stock video.
musicbooltrueBackground music based on detected mood.

sfxbooltrueTransition sound effects (whoosh, pop, swipe).
zoombooltrueDynamic camera zoom-in at high-impact moments.
silencebooltrueAuto-remove silent gaps between sentences.
jumpcutbooltrueJump-cuts at energetic moments for punchy pacing.
facetrackbooltrueSmart 9:16 reframe with face tracking.
viralbooltrueExtract best 7-12s hook clip as viral_video_path.
backgroundboolfalseAI virtual studio background replacement.
logoboolfalseOverlay brand logo (upload separately first).
video_qualitystring"1080p"Output resolution: "720p" or "1080p".

Example Request Body

Response (200 OK)

{ "success": true, "job_id": "a1b2c3d4e5f6g7h8", "status": "transcribing", "message": "UGC video processing started in the background." }
📊 Get Job Status

Poll this endpoint every 5–10 seconds. When status is "completed", download links are ready.

GET /api/ugc/job/{job_id} Get status and download links ▾

Response Fields

FieldTypeDescription

progressintPercentage complete (0–100).
result_video_pathstringURL to the final AI-edited video. CDN URL if R2 is configured, otherwise local path.
viral_video_pathstringURL to the 7-12s Shorts hook clip (only when viral=true).

error_messagestring|nullError details when status is "failed".

🔄 Job Status Flow

A job transitions through these states during processing.

StatusDescription
pendingVideo uploaded, waiting for /process call.
transcribingWhisper AI transcribing speech to words with timestamps.
processingAll AI stages running: zoom, B-roll gen, reframe, captions, music mix.
completedDone. result_video_path and viral_video_path are available.
failedPipeline error. Check error_message for details.
🚨 Error Codes

All errors return JSON with a detail field describing the issue.

HTTP CodeMeaning
401Missing or invalid X-App-Token header.
400Bad request — file too large, invalid format, or job already running.
404Job not found or result not ready yet.
500Internal server error — retry after a few seconds.
💻 Code Examples

Full end-to-end examples — upload → process → poll → download.

cURL

# Step 1: Upload curl -X POST https://your-server.com/api/ugc/upload \ -H "X-App-Token: YOUR_TOKEN" \ -F "file=@/path/to/video.mp4" # Step 2: Process (replace JOB_ID with value from step 1) curl -X POST https://your-server.com/api/ugc/process/JOB_ID \ -H "X-App-Token: YOUR_TOKEN" \ -H "Content-Type: application/json" \ -d '{"caption":true,"subtitle_style":"two_line_zoom_in","broll":true,"music":true,"bgm_mood":"Motivational","viral":true}' # Step 3: Poll (repeat until status = "completed") curl https://your-server.com/api/ugc/job/JOB_ID \ -H "X-App-Token: YOUR_TOKEN"

Python

import requests, time BASE = "https://your-server.com" HDR = {"X-App-Token": "YOUR_TOKEN"} # 1. Upload with open("video.mp4", "rb") as f: job_id = requests.post(f"{BASE}/api/ugc/upload", headers=HDR, files={"file": f}).json()["job_id"] # 2. Process requests.post(f"{BASE}/api/ugc/process/{job_id}", headers=HDR, json={ "caption": True, "subtitle_style": "two_line_zoom_in", "broll": True, "music": True, "bgm_mood": "Motivational", "viral": True, "video_quality": "1080p" }) # 3. Poll until done while True: job = requests.get(f"{BASE}/api/ugc/job/{job_id}", headers=HDR).json() print(f"Status: {job['status']} ({job['progress']}%)") if job["status"] == "completed": print("Done!", job["result_video_path"]) break elif job["status"] == "failed": print("Error:", job["error_message"]); break time.sleep(8)

JavaScript (Fetch)

const BASE = "https://your-server.com"; const HDR = { "X-App-Token": "YOUR_TOKEN" }; async function processVideo(file) { // 1. Upload const form = new FormData(); form.append("file", file); const { job_id } = await ( await fetch(`${BASE}/api/ugc/upload`, { method:"POST", headers:HDR, body:form }) ).json(); // 2. Process await fetch(`${BASE}/api/ugc/process/${job_id}`, { method:"POST", headers:{...HDR,"Content-Type":"application/json"}, body: JSON.stringify({ caption:true, subtitle_style:"two_line_zoom_in", broll:true, music:true, bgm_mood:"Motivational", viral:true }) }); // 3. Poll return new Promise((resolve, reject) => { const t = setInterval(async () => { const job = await (await fetch(`${BASE}/api/ugc/job/${job_id}`,{headers:HDR})).json(); console.log(job.status, job.progress + "%"); if (job.status === "completed") { clearInterval(t); resolve(job); } else if (job.status === "failed") { clearInterval(t); reject(job.error_message); } }, 8000); }); }
🎞️ B-roll Sources Guide

Set broll_source in your process request to control how B-roll visuals are generated.

ValueWhat it doesRequirement
pollinations 🤖 Auto-generates cinematic AI images via Flux model (Pollinations.ai). No API key needed. Default. None
pexels 🎬 Downloads real stock videos from Pexels API matching keywords from transcript. Portrait (9:16) preferred. PEXELS_API_KEY in .env
meta_ai 📤 Uses images you pre-uploaded via the dashboard Meta AI Extension. You manually generate + upload images first. Pre-upload via dashboard
ℹ️
Pexels Setup: Get a free API key at pexels.com/api. Add it to your .env file as PEXELS_API_KEY=your_key_here then restart the server.

Example — Use Pexels Stock Videos

curl -X POST https://your-server.com/api/ugc/process/JOB_ID \ -H "X-App-Token: YOUR_TOKEN" \ -H "Content-Type: application/json" \ -d '{ "broll": true, "broll_source": "pexels", "caption": true, "music": true }'

Example — Use Pollinations AI Images (default)

curl -X POST https://your-server.com/api/ugc/process/JOB_ID \ -H "X-App-Token: YOUR_TOKEN" \ -H "Content-Type: application/json" \ -d '{"broll": true, "broll_source": "pollinations"}'
🌆 AI Studio Background

Replace the original video background with an AI-generated studio scene using MediaPipe segmentation + Flux AI image generation. No green screen required.

⚠️
Background replacement requires MediaPipe to be installed on the server (pip install mediapipe). The server must also have sufficient GPU/CPU for real-time segmentation.
💡
You can write any descriptive text as background_style — it's sent as a Flux AI image prompt. The more specific, the better the result.

Process Request Fields for Background

FieldTypeDefaultDescription
backgroundboolfalseEnable AI background replacement. Must be true to activate.
background_stylestringsee belowFlux AI prompt describing the background scene. Any descriptive text works.

🏷️ Brand Logo Overlay

Overlay your brand logo on every frame of the final video. Requires two API calls: first upload the logo file, then enable it in the process request.

ℹ️
The logo upload endpoint stores the logo against the job. Then when logo: true is set in the process request, the pipeline automatically loads and overlays it on every frame.

Step 1 — Upload Logo File

POST /api/ugc/upload-logo/{job_id} Upload brand logo image ▾
Content-Type
multipart/form-data
Accepted Formats
.png, .jpg, .webp
💡
Use a PNG with transparent background for best results — the transparency is preserved when overlaying on video frames.

Request Body (form-data)

FieldTypeRequiredDescription
fileFileRequiredLogo image file (.png recommended with transparent bg).

cURL Example

curl -X POST https://your-server.com/api/ugc/upload-logo/JOB_ID \ -H "X-App-Token: YOUR_TOKEN" \ -F "file=@/path/to/your-logo.png"

Response (200 OK)

{ "success": true, "path": "/uploads/ugc/JOB_ID/brand_logo.png" }

Step 2 — Enable Logo in Process Request

After uploading, include "logo": true in your process call.

cURL — Full Logo + Process Example

# Step 1: Upload logo curl -X POST https://your-server.com/api/ugc/upload-logo/JOB_ID \ -H "X-App-Token: YOUR_TOKEN" \ -F "file=@logo.png" # Step 2: Process with logo enabled curl -X POST https://your-server.com/api/ugc/process/JOB_ID \ -H "X-App-Token: YOUR_TOKEN" \ -H "Content-Type: application/json" \ -d '{ "logo": true, "caption": true, "broll": true, "music": true, "viral": true }'

Python — Full Logo + Process Example

import requests BASE = "https://your-server.com" HDR = {"X-App-Token": "YOUR_TOKEN"} # 1. Upload video with open("video.mp4", "rb") as f: job_id = requests.post(f"{BASE}/api/ugc/upload", headers=HDR, files={"file": f}).json()["job_id"] # 2.
