##  Excel Importer Parser Mapping Fix & DB Cleanup

### 1. Parser Mapping Fix
* **The Problem:** When importing an Excel sheet containing headers but omitting a "subtopic" column, the parser fell back to default index settings. This caused column 2 ("Video Length") to be parsed as the subtopic name, creating invalid subtopics like "120 sec" or "90 sec".
* **The Solution:** Modified the header matching block inside `/classroom/subjects/{subject_id}/upload-index` (`app/routes/classroom.py`):
  * When a header row is detected, we now initialize all column indexes to `-1` so mappings are strictly defined by headers.
  - Required fields (`chapter_col` and `topic_col`) default to `0` and `1` if not found in the headers, while optional fields (`subtopic_col`, `script_col`, and `length_col`) remain at `-1`.
  * This successfully blocks column 2 from being misclassified as a subtopic, mapping video lengths and scripts directly onto the Topic level instead.

### 2. Database Cleanup
* Executed a cleanup script to delete invalid subtopic records generated by previous imports:
  ```python
  deleted = db.query(SubtopicClassroom).filter(SubtopicClassroom.name.like('%sec%')).delete()
  db.commit()
  ```
  *Result:* Successfully removed **110** corrupted subtopics from the database.
