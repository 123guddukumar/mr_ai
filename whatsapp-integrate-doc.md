# MR AI RAG — WhatsApp Integration Guide

This guide explains how to connect your configured AI Agents (with active RAG/Vector Database capabilities) directly into any WhatsApp automation platform, gateway, or Meta Cloud API account.

---

## 🛠️ 1. Integration Architectures

You can integrate your AI Agent in two ways:

### Architecture A: Headless API Call (Recommended)
If you already use an external WhatsApp platform (like WATI, ManyChat, Chatfuel, or a custom script), you do not need to configure webhooks or credentials on this backend. 
When your WhatsApp system receives a user query, it simply calls our FastAPI Agent API, gets the answer, and routes it back.

```
[User on WhatsApp] ──> [WhatsApp gateway (WATI/ManyChat)] 
                                   │
                                   ▼  (POST Request)
                         [FastAPI Backend /api/agents/.../public-ask]
                                   │
                                   ▼  (Returns JSON Answer)
[User on WhatsApp] <── [WhatsApp gateway (WATI/ManyChat)]
```

### Architecture B: Direct Meta Webhook Integration
If you want our FastAPI server to communicate directly with Meta's servers, you configure your Meta Webhook URL to point to this server and save the Meta credentials in your `.env` file.

## 🚀 2. Architecture A: Headless API Reference

To query any AI Agent, make a `POST` request to the public endpoint.

### Request Endpoint
```http
POST /api/agents/{agent_id}/public-ask
Content-Type: application/json
```

* **URL Parameters**:
  * `{agent_id}`: The unique identifier of your agent (e.g. `da3243d9babb9387`).

### Request Payload (JSON)
```json
{
  "question": "What is the syllabus of BPSC Exam?",
  "session_id": "wa_919876543210",
  "device_id": "919876543210",
  "device_name": "WhatsApp Client"
}
```

* **Field Descriptions**:
  * `question` *(String, Required)*: The text query sent by the user on WhatsApp.
  * `session_id` *(String, Required)*: A unique identifier for the user session (prefixing with `wa_` followed by the sender's phone number is recommended). This keeps the chat history context alive for subsequent questions.
  * `device_id` *(String, Required)*: The sender's phone number. Used to map visitor logs.
  * `device_name` *(String, Optional)*: Descriptive device tag.

### Response Payload (JSON)
```json
{
  "answer": "The BPSC exam consists of three stages: Prelims, Mains, and the Interview...",
  "sources": [
    {
      "source_file": "BPSC_Guide.pdf",
      "page_number": 3
    }
  ],
  "is_rag": true,
  "session_id": "wa_919876543210"
}
```

* **Response Fields**:
  * `answer`: The text generated by the AI agent based on instructions and indexed knowledge.
  * `is_rag`: Boolean showing if the answer was retrieved from the database context documents.

### Code Examples for Developers

#### 🐍 Python Integration Example
```python
import requests

API_URL = "http://127.0.0.1:8000/api/agents/da3243d9babb9387/public-ask"
payload = {
    "question": "Hello! Can you help me?",
    "session_id": "wa_919876543210",
    "device_id": "919876543210",
    "device_name": "WhatsApp Python Script"
}

response = requests.post(API_URL, json=payload)
if response.status_code == 200:
    data = response.json()
    print("AI Answer:", data["answer"])
else:
    print("Error:", response.status_code, response.text)
```

#### 🟨 JavaScript / Node.js Fetch Example
```javascript
const agentId = "da3243d9babb9387";
const backendUrl = "https://your-server-domain.com";

async function getAgentAnswer(userMessage, userPhone) {
  try {
    const response = await fetch(`${backendUrl}/api/agents/${agentId}/public-ask`, {
      method: "POST",
      headers: {
        "Content-Type": "application/json"
      },
      body: JSON.stringify({
        question: userMessage,
        session_id: `wa_${userPhone}`,
        device_id: userPhone,
        device_name: "WhatsApp Client"
      })
    });
    
    const data = await response.json();
    return data.answer; // Send this back to user
  } catch (error) {
    console.error("API Call Failed:", error);
    return "Sorry, I am facing connectivity issues.";
  }
}
```

## 🔗 3. Architecture B: Direct Webhook Integration Reference

If you are setting up the Meta App console yourself, use these parameters:

### Webhook Verification (Handshake)
* **Method**: `GET`
* **URL**: `https://<your-public-server>/api/whatsapp/webhook`
* **Verify Token**: `mr_ai_rag_verify_token` (as configured in `.env`)
* **Handshake Parameter Mode**: Select **`subscribe`** on Meta Console.

### Event Notification (Message updates)
* **Method**: `POST`
* **URL**: `https://<your-public-server>/api/whatsapp/webhook`
* **Subscription Fields**: Select and subscribe to **`messages`** in Meta developers dashboard under Webhook Fields.

### Required environment variables in `.env`:
```ini
WHATSAPP_PHONE_NUMBER_ID=your_meta_phone_number_id
WHATSAPP_ACCESS_TOKEN=your_meta_system_access_token
WHATSAPP_VERIFY_TOKEN=mr_ai_rag_verify_token
WHATSAPP_DEFAULT_AGENT_ID=da3243d9babb9387
```
