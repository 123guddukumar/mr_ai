🚀 Introduction

MR AI RAG (Retrieval-Augmented Generation) is an AI document intelligence platform. Upload PDFs, YouTube videos, websites, or video files — then ask questions in natural language and get AI-powered answers with citations.

✨ Core Capabilities
  • Multi-source ingestion — PDF, YouTube, URL/website scraping, MP4/WAV transcription
  • Local embeddings — Uses sentence-transformers locally, no embedding API key needed
  • Multi-provider LLM — OpenAI, Google Gemini, Anthropic Claude, Ollama (local), HuggingFace
  • Client authentication — Register, email OTP verification, personal API key management
  • Chat history — All conversations saved to your dashboard automatically
  • FAISS vector store — Fast semantic search across all indexed content

⚡ Quick Start (5 min)

  1. Register — Go to /login → Create Account → enter your name, email, password → a 6-digit OTP is emailed to you.
  2. Verify email — Enter the OTP code from your email. Your account is created and you're redirected to the Dashboard.
  3. Generate API key — In the Dashboard, find "Generate API Key", enter a label, click ⚡ Generate. Copy the key immediately — it's shown only once.
  4. Configure LLM — In the Playground, select your AI provider (e.g. Gemini), paste your API key, click ⚡ Apply Provider.
  5. Upload a source — In the Playground left panel, upload a PDF or paste a YouTube URL.
  6. Ask a question — Type your question in the chat input and press Enter. AI answers with source citations.
  7. Check history — Visit your Dashboard → Chat History tab to see all conversations.

Render PostgreSQL Setup

  1. Go to dashboard.render.com → New → PostgreSQL
  2. Choose Free plan → Create Database
  3. In the DB dashboard, click Connect → copy the External Database URL
  4. Paste it as DATABASE_URL in your .env
  5. Restart the server — tables are auto-created on startup
ℹ️
Render free PostgreSQL databases expire after 90 days. Upgrade to paid for production use.

👤 Register & Login

MR AI RAG uses a client-based authentication system. Every client gets a unique client_id and a session token.

Via Web UI (recommended)

Visit /login, switch to "Create Account", fill the form, and verify your email with the OTP code sent to you.

Via API

# Step 1: Send OTP
curl -X POST /api/clients/send-otp \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","name":"Your Name","purpose":"register"}'

# Step 2: Register with OTP
curl -X POST /api/clients/register \
  -H "Content-Type: application/json" \
  -d '{"name":"Your Name","email":"you@example.com","password":"secret123","otp":"123456"}'

Login

curl -X POST /api/clients/login \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","password":"secret123"}'

The response contains client_id and token. Store these securely — token is your X-Client-Token for authenticated requests.

📩 Email OTP Verification

Email verification is required for registration and password reset. OTPs are valid for 10 minutes and expire after use.

⚠️
Make sure your Gmail account uses an App Password, not your regular Gmail password. Go to Google Account → Security → 2-Step Verification → App Passwords.

Forgot Password

  1. Go to Dashboard → click the ⚙️ gear icon → Forgot Password
  2. Enter your email address → click Send OTP
  3. Check your email, enter the 6-digit code
  4. Enter your new password → click Reset Password
# Via API
curl -X POST /api/clients/forgot-password \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com"}'

curl -X POST /api/clients/reset-password \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","otp":"123456","new_password":"newPass123"}'

🔑 API Keys

API keys authenticate requests to protected endpoints. Use the X-API-Key header on every upload and query request.

🚨
The full API key is shown only once at generation time. Store it immediately in a secure place — it cannot be retrieved again.

Generate via Dashboard

  1. Go to Dashboard
  2. Find ⚡ Generate API Key in the left panel
  3. Enter a label (e.g. "My App")
  4. Click ⚡ Generate — copy the key shown

Generate via API

# Use your client token as X-Admin-Secret
curl -X POST /api/keys/generate \
  -H "Content-Type: application/json" \
  -H "X-Admin-Secret: clt-your-token-here" \
  -d '{"name":"My App"}'

Using API Keys

# Add X-API-Key header to every request
curl -X POST /api/upload \
  -H "X-API-Key: mrairag-your-api-key-here" \
  -F "file=@document.pdf"

Revoke Keys

In the Dashboard → My API Keys panel, click Revoke next to any active key. Revoked keys immediately stop working.

📄 Uploading Sources

Sources are split into chunks, embedded, and stored in a FAISS vector index. All queries search across all indexed chunks.

PDF Upload

curl -X POST /api/upload \
  -H "X-API-Key: mrairag-..." \
  -F "file=@report.pdf"
# Returns: {success, filename, total_pages, total_chunks}

YouTube Video

curl -X POST /api/ingest-youtube \
  -H "Content-Type: application/json" \
  -H "X-API-Key: mrairag-..." \
  -d '{"url":"https://youtube.com/watch?v=...","whisper_model":"base"}'
# Returns: {success, title, total_chunks, word_count, method_used}

Website / URL

curl -X POST /api/ingest-url \
  -H "Content-Type: application/json" \
  -H "X-API-Key: mrairag-..." \
  -d '{"url":"https://example.com/article"}'

Video / Audio File

curl -X POST "/api/ingest-video?whisper_model=base" \
  -H "X-API-Key: mrairag-..." \
  -F "file=@lecture.mp4"
ℹ️
Whisper models: tiny (fastest), base, small, medium, large (most accurate). Larger models require more RAM and take longer.

💬 Asking Questions

curl -X POST /api/ask \
  -H "Content-Type: application/json" \
  -H "X-API-Key: mrairag-..." \
  -d '{
    "question": "What are the main conclusions?",
    "top_k": 5
  }'
# Returns: {answer, sources, context_found, model_used}
ParameterTypeDescription
questionstringYour question (required)
top_kintegerNumber of context chunks to retrieve (default: 5)
✅
The AI only answers from indexed content. If no relevant content is found, it says so rather than hallucinating.

🔍 Troubleshooting

Chat history not showing on dashboard

Make sure you're logged in when using the Playground — messages are saved using your session token from localStorage. If you're not logged in, messages aren't saved. Log in at /login then use the Playground.

OTP email not received

  • Check your spam/junk folder
  • Ensure SMTP_PASSWORD is a Gmail App Password (not your regular password)
  • Enable "2-Step Verification" then generate an App Password at myaccount.google.com/apppasswords
  • OTP is valid for only 10 minutes — click "Resend OTP" if expired

Database connection error on startup

  • Check DATABASE_URL in .env — must use PostgreSQL format
  • For Render: use the External connection URL (not internal)
  • The server will still start even if DB fails — it logs a warning and continues

API key rejected (401)

  • Ensure header is exactly X-API-Key
  • Key may have been revoked — generate a new one from Dashboard
  • If API_KEYS_ENABLED=false, no key is needed

Provider not responding

  • Verify your provider API key is correct and has credits
  • For Ollama: ensure ollama serve is running locally
  • Check /api/health to see current provider status

❓ FAQ

Is my data stored permanently?

Uploaded documents are indexed in the FAISS vector store on the server. Chat history is stored in PostgreSQL. You can clear your chat history anytime from the Dashboard.

Can I use multiple AI providers?

Yes! Switch between OpenAI, Gemini, Claude, Ollama, and HuggingFace at any time via the Playground's AI Provider panel or via POST /api/provider/config. The change applies immediately to all subsequent queries.

What's the difference between my Client ID and API Key?

  • Client ID — your unique identity identifier (e.g. client-a1b2c3d4e5). Used as X-Admin-Secret when generating API keys.
  • Session Token — your login credential (e.g. clt-...). Used as X-Client-Token for profile/history/notification requests. Also works as X-Admin-Secret.
  • API Key — used as X-API-Key for all document upload and query requests.

Can I share my API key?

You can share API keys for integrations, but be careful — anyone with your key can upload and query documents. Revoke keys immediately if compromised.

Is there a rate limit?

No built-in rate limits are applied by default. For production deployments, consider adding rate limiting via a reverse proxy (nginx) or FastAPI middleware.

How do I delete my account?

Account deletion is not currently available through the UI. Contact the admin or use DELETE /api/clients/me (coming soon).
