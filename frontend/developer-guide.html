1 What is MR AI RAG?

MR AI RAG (My RAG AI) is a platform that lets you build AI chatbots powered by your own knowledge base. Upload PDFs, websites, YouTube videos — the AI answers questions using only your content.

📄 Upload sources → PDF, website URL, YouTube video, JSON
↓
🧠 AI indexes content → chunks text, creates embeddings in vector DB
↓
❓ User asks question → system finds relevant chunks (RAG)
↓
🤖 LLM generates answer → using only your knowledge base content
↓
🔊 Optional TTS → answer spoken aloud (voice-to-voice mode)

2 Quick Start (5 Minutes)

1Get an API Key
Register at https://yourdomain.com → Dashboard → Settings → API Keys → Generate Key.
You'll get a key like: mrairag-k-xxxxxxxxxxxxxxxxxx
2Login & Get Client Token
# Python example
import requests

BASE = "https://yourdomain.com"
HEADERS = {"X-API-Key": "mrairag-k-YOUR_KEY"}

# Login
r = requests.post(f"{BASE}/api/clients/login",
    json={"email": "you@example.com", "password": "yourpassword"},
    headers=HEADERS)
token = r.json()["token"]
CLIENT_HEADERS = {**HEADERS, "X-Client-Token": token}
3Create Memory, Add PDF, Ask Question
# Create a memory (chatbot)
mem = requests.post(f"{BASE}/api/memory",
    json={"name": "Product FAQ", "description": "Our product knowledge base"},
    headers=CLIENT_HEADERS).json()
memory_id = mem["memory_id"]

# Upload a PDF
with open("product_manual.pdf", "rb") as f:
    requests.post(f"{BASE}/api/memory/{memory_id}/upload-pdf",
        files={"file": f}, headers=CLIENT_HEADERS)

# Ask a question
answer = requests.post(f"{BASE}/api/memory/{memory_id}/ask",
    json={"question": "What is the return policy?"},
    headers=CLIENT_HEADERS).json()
print(answer["answer"])

3 Authentication

All API requests need either:

  • X-API-Key — for server-to-server calls (your API key)
  • X-Client-Token — for user-specific endpoints (login to get this)
# Always include both headers for client requests
headers = {
    "X-API-Key": "mrairag-k-...",
    "X-Client-Token": "clt-...",
    "Content-Type": "application/json"
}

6 Ask Questions via API

POST /api/memory/{memory_id}/ask

Body:
{
  "question": "What are the features?",
  "top_k": 3,        // how many source chunks to use (default: 3)
  "history": [       // optional: last N turns for context
    {"role": "user", "content": "previous question"},
    {"role": "assistant", "content": "previous answer"}
  ]
}

Response:
{
  "answer": "The key features are...",
  "sources": ["manual.pdf", "website.com"],
  "latency_ms": 1240
}

Multi-language Support

# Force a specific language in response
{
  "question": "क्या return policy है?",  // Hindi question
  "language": "hindi"                    // force Hindi response
}

7 Streaming Responses (SSE)

For real-time word-by-word responses, use the streaming endpoint. First token appears in ~300ms.

# Python with httpx (async streaming)
import httpx, asyncio

async def stream_answer():
    async with httpx.AsyncClient() as client:
        async with client.stream("POST",
            f"{BASE}/api/memory/{MEMORY_ID}/ask",
            json={"question": "Tell me everything about the product", "stream": True},
            headers=CLIENT_HEADERS, timeout=120) as resp:
            async for line in resp.aiter_lines():
                if line.startswith("data: "):
                    chunk = line[6:]
                    if chunk != "[DONE]":
                        print(chunk, end="", flush=True)

asyncio.run(stream_answer())
JavaScript (EventSource)
const evtSource = new EventSource(
  `/api/memory/${MEMORY_ID}/stream-ask?question=Hello&token=${CLIENT_TOKEN}`
);
evtSource.onmessage = (e) => {
  if (e.data === '[DONE]') { evtSource.close(); return; }
  document.getElementById('answer').textContent += e.data;
};

8 Voice System Integration

MR AI RAG supports three TTS (Text-to-Speech) engines for voice responses:

EngineKeyLanguagesBest For
MRAI (Browser)Free30+ via Web Speech APIMultilingual, offline
ElevenLabsAPI keyEnglish (best quality)Ultra-realistic English voices
Sarvam AIAPI keyHindi, Tamil, Telugu, 9+ IndianIndian languages

Voice-to-Voice Flow

🎤 User speaks → Web Speech API (STT) transcribes to text (~200-400ms)
↓
🔍 Text sent to RAG API → vector search + LLM answer (~800-1500ms)
↓
🔊 Answer text → TTS engine → audio played (~300-600ms)
↓
🎤 Loop: system listens again automatically

JavaScript Voice Integration Example

// Speech-to-Text (STT) using Web Speech API
const SR = window.SpeechRecognition || window.webkitSpeechRecognition;
const rec = new SR();
rec.lang = 'en-IN';  // or 'hi-IN', 'ta-IN', etc.
rec.onresult = async (e) => {
  const question = e.results[0][0].transcript;

  // Send to API
  const res = await fetch(`/api/memory/${MEMORY_ID}/ask`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json', 'X-Client-Token': TOKEN },
    body: JSON.stringify({ question })
  });
  const { answer } = await res.json();

  // Text-to-Speech
  const u = new SpeechSynthesisUtterance(answer);
  u.lang = 'en-IN';
  speechSynthesis.speak(u);
};
rec.start();
Tip: Use language detection to automatically pick the right STT/TTS language. Check unicode ranges: Devanagari (Hindi) is \u0900-\u097F, Tamil is \u0B80-\u0BFF.

11 Full JavaScript SDK Example

// MR AI RAG SDK — drop into any webpage
class MRaiRag {
  constructor(memoryId, clientToken, baseUrl = '') {
    this.memoryId = memoryId;
    this.headers = {
      'Content-Type': 'application/json',
      'X-Client-Token': clientToken
    };
    this.base = baseUrl;
  }

  async ask(question, history = []) {
    const r = await fetch(`${this.base}/api/memory/${this.memoryId}/ask`, {
      method: 'POST', headers: this.headers,
      body: JSON.stringify({ question, top_k: 3, history })
    });
    return r.json(); // { answer, sources, latency_ms }
  }

  async addUrl(url) {
    const r = await fetch(`${this.base}/api/memory/${this.memoryId}/add-url`, {
      method: 'POST', headers: this.headers,
      body: JSON.stringify({ url })
    });
    return r.json();
  }

  async getHistory(limit = 10) {
    const r = await fetch(
      `${this.base}/api/memory/${this.memoryId}/chat-history?limit=${limit}`,
      { headers: this.headers }
    );
    return r.json();
  }
}

// Usage
const bot = new MRaiRag('mem-abc123', 'clt-yourtoken');
const { answer } = await bot.ask('What is your refund policy?');
console.log(answer);

12 Error Handling Best Practices

try {
  const res = await fetch(url, { method: 'POST', ... });
  if (!res.ok) {
    const err = await res.json();
    // err.detail contains the error message
    if (res.status === 401) console.error('Invalid token — re-login');
    if (res.status === 404) console.error('Memory not found');
    if (res.status === 429) console.error('Rate limited — retry in 60s');
    throw new Error(err.detail || 'Request failed');
  }
  return res.json();
} catch (e) {
  // Handle network errors
  console.error('API error:', e.message);
}
Rate Limits: Default is 60 requests/minute per API key. For higher limits, contact the admin.
Best Practice: Cache the client token in localStorage after login. It expires after 30 days. Check for 401 errors and prompt re-login.
