2 Prerequisites

  • AWS Account with billing enabled
  • Domain name (optional but recommended, e.g. from Route 53 or GoDaddy)
  • AWS CLI installed and configured: aws configure
  • Your project code pushed to GitHub
  • Gemini API key and/or OpenAI API key

3 Create RDS PostgreSQL Database

1Create RDS Instance
Go to AWS Console → RDS → Create database. Choose:
  • Engine: PostgreSQL 15
  • Template: Free Tier (or Production)
  • DB instance identifier: mrairag-db
  • Master username: mrairag (save this password!)
  • Instance class: db.t3.micro
  • Storage: 20 GB gp3
  • VPC: Default (remember which one)
  • Public access: No (access only from EC2)
2Configure Security Group for RDS
After creation, go to the RDS instance → Security Group → Add Inbound Rule:
Type: PostgreSQL (5432), Source: EC2 Security Group (not 0.0.0.0/0).
3Get the Connection Endpoint
Copy the RDS endpoint (looks like: mrairag-db.xxxx.us-east-1.rds.amazonaws.com). You'll use this as your DATABASE_URL.

4 Launch EC2 Server

1Launch EC2 Instance
Go to EC2 → Launch Instance:
  • AMI: Ubuntu 22.04 LTS
  • Instance type: t3.medium (2 vCPU, 4 GB RAM — minimum for embedding model)
  • Key pair: Create or select existing .pem key
  • Security Group: Allow ports 22 (SSH), 80 (HTTP), 443 (HTTPS)
  • Storage: 30 GB gp3
  • IAM Role: Create role with S3FullAccess and SecretsManagerReadWrite
2Connect via SSH
# Replace with your .pem file and EC2 public IP
chmod 400 your-key.pem
ssh -i your-key.pem ubuntu@YOUR_EC2_PUBLIC_IP
3Install System Dependencies
sudo apt update && sudo apt upgrade -y
sudo apt install -y python3.11 python3.11-venv python3-pip git nginx certbot python3-certbot-nginx postgresql-client

5 Deploy the Application

1Clone the Repository
git clone https://github.com/YOUR_USERNAME/mr_ai_rag_v2.git /opt/mrairag
cd /opt/mrairag
2Create Python Virtual Environment
python3.11 -m venv venv
source venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
3Create .env File
# /opt/mrairag/.env  — NEVER commit this file to git!
DATABASE_URL=postgresql://mrairag:PASSWORD@mrairag-db.xxxx.us-east-1.rds.amazonaws.com/mrairag
SECRET_KEY=your-64-char-random-secret
GEMINI_API_KEY=AIza...
OPENAI_API_KEY=sk-...         # optional
GOOGLE_CLIENT_ID=xxxxx.apps.googleusercontent.com
ADMIN_SECRET=your-admin-secret-key
S3_BUCKET=mrairag-files
AWS_REGION=us-east-1
4Test Run
cd /opt/mrairag
source venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8000
# Visit http://YOUR_IP:8000 to verify it works, then Ctrl+C

6 Nginx + HTTPS with Let's Encrypt

1Configure Nginx
sudo nano /etc/nginx/sites-available/mrairag
Paste this config:
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        client_max_body_size 100M;    # for PDF/video uploads
        proxy_read_timeout 300s;      # for long AI responses
    }
}

sudo ln -s /etc/nginx/sites-available/mrairag /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
2Get Free SSL Certificate
# Point your domain's A record to EC2 public IP FIRST
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
# Follow prompts — it auto-configures HTTPS + auto-renewal

8 Process Management with systemd

1Create systemd Service
sudo nano /etc/systemd/system/mrairag.service
Paste:
[Unit]
Description=MR AI RAG FastAPI Service
After=network.target

[Service]
User=ubuntu
WorkingDirectory=/opt/mrairag
EnvironmentFile=/opt/mrairag/.env
ExecStart=/opt/mrairag/venv/bin/gunicorn app.main:app \
  -w 2 -k uvicorn.workers.UvicornWorker \
  --bind 127.0.0.1:8000 \
  --timeout 300 \
  --access-logfile /var/log/mrairag/access.log \
  --error-logfile /var/log/mrairag/error.log
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
2Enable & Start Service
sudo mkdir -p /var/log/mrairag
sudo systemctl daemon-reload
sudo systemctl enable mrairag
sudo systemctl start mrairag
sudo systemctl status mrairag    # check it's running

9 S3 for File Storage

1Create S3 Bucket
aws s3 mb s3://mrairag-files --region us-east-1
2Block Public Access (Security)
aws s3api put-public-access-block \
  --bucket mrairag-files \
  --public-access-block-configuration "BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true"
3Attach IAM Role (Preferred over Access Keys)
Attach the EC2 IAM Role created in Step 4.1 that has AmazonS3FullAccess. This way you don't store AWS credentials in the .env file.

10 Security Hardening

  • Never expose RDS publicly (always use Private VPC)
  • Use AWS Secrets Manager instead of plaintext .env for production API keys
  • Enable AWS WAF on CloudFront to block SQL injection & XSS attacks
  • Enable RDS encryption at rest (toggle during creation)
  • Use Security Groups as firewalls — only open ports 80, 443 (and 22 to your IP only)
  • Enable EC2 CloudWatch monitoring for CPU/memory alerts
  • Rotate API keys every 90 days using Secrets Manager rotation
  • Enable S3 versioning for backup: aws s3api put-bucket-versioning --bucket mrairag-files --versioning-configuration Status=Enabled
⚠️ Critical: Add .env to your .gitignore immediately. Never push secrets to GitHub.

12 CI/CD Pipeline (GitHub Actions)

1Create .github/workflows/deploy.yml
name: Deploy to AWS EC2

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Deploy via SSH
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.EC2_HOST }}
          username: ubuntu
          key: ${{ secrets.EC2_SSH_KEY }}
          script: |
            cd /opt/mrairag
            git pull origin main
            source venv/bin/activate
            pip install -r requirements.txt
            sudo systemctl restart mrairag
2Add GitHub Secrets
In your GitHub repo → Settings → Secrets → Actions, add:
  • EC2_HOST — your EC2 public IP or domain
  • EC2_SSH_KEY — contents of your .pem file
Every push to main will now auto-deploy to AWS! ✅
